Commands

nginx-dev has a rich yet basic command line functionality, here is the output of --help:

Usage: ./nginx-dev.sh COMMAND [options]

Main commands:

 start                                                starts the nginx environment
 stop                                                stops the nginx environment
 sites                                               main site command
    list                                               list all the sites under the environment
    add [www.example.dev]         adds a new site under the config
    remove [www.example.dev]  removes a website
    clean [www.example.dev]      rebuilds a sites configuration to repo default

 config                                            main global config command
    list                                               lists all the global variables
    set VAR="var"                           sets a variable in the global config, leave "" for nothing

 rebuild-env                                  rebuilds the development environment from the selected repo
 test-nginx-config                         tests the current root nginx config

Options:

 --help - returns this screen
 --repo=[git url] - overrides the global repo
 --root=[location - overrides the global root location
 --env="var:test, var2:test2" - extra variables


Here is this repo's layout and functions:
  • Root Directory:
    • configs: The main nginx configuration folder
      • nginx.conf - The main Nginx config file
      • default-website.conf - A generic site config file
      • per-site: - per site config folder
        • *.conf - a per site config, typically formatted with siten name (ex. localhost.conf)
    • websites: - Our site data
    • main.conf - Repo configuration file
    • startup.sh - An optional configuration script

Configuration Files

There are a few things to settle before getting into the configuration files. nginx-dev has a simple variable system that dictate how variables are passed for parsing.

Variable Type Description
Command Line Variables issued in the command line, these variables are looked at first
Global These variables are loaded in ~/.nginx-dev.conf; These variables are looked at second and can be overridden by the Command Line
Repo Loaded in the Root environment's main.conf file, These variables will be overridden by the Command Line and Global variables

As a side note, all variables are unique within the script, starting with CLI_, GLOBAL_, and REPO_. But when templating nginx config files, these identifiers are ignored and comes as first come first serve basis.

main.conf

main.conf is the repo configuration file for all the variables, Here is this repo's config

# Variables for the repo, can include anything, but these are the required variables
# Once Processed, these variables become REPO_
# Blank lines are not permitted, if you wish to make things neater, add # to a new lines
# Do NOT add # on the variable lines, you will screw the system up.
#
# Main Config for nginx
NGINXCONFIG=configs/nginx.conf
#
# Default Site Config
DEFAULTCONFIG=configs/default-website.conf
#
# Extra directories that need to be created (not copied by git)
MKDIR=run, tmp, tmp/client_body, tmp/fastcgi_temp, tmp/proxy_temp, tmp/scgi_temp, tmp/uwsgi_temp, logs
#
# Main script for primary setup
PRIMARYSCRIPT=setup.sh
